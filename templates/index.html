Usage de l'api yggnode

Serveur de synchronisation des flux rss et .torrents pour yggtorrent.li

Le site étant sous protection cloudflare, le projet a pour but de rétablir l'utilisation de ses flux rss :

  • Pour cela il utilise le projet flarrsolverr afin de résoudre le challenge cloudflare et synchronisé régulièrement les flux rss et leur .torrent associez sur le serveur.
  • Une api est ensuite mise a disposition pour lire les flux et dll les .torrent directement depuis le serveur.

Pour les flux rss

Example : {{ server }}/rss?id=2139&passkey=Ttw5xQcCbmjHUwC9jCs8fdbrGEnF8yEt

Deux arguments sont nécessaire dans l'url :

  • ID du flux : Liste des flux synchronisé sur ce serveur
  • Passkey personnelle : vous trouverait votre passkey sur votre compte ygg ici, dans la rubrique INFORMATIONS RELATIVES AU TRACKER

Pour facilité l'usage, vous pouvez utilisé le générateur de lien : LINKS GENERATOR

  • renseigné votre passkey
  • Une foi généré copié les lien dans votre client torrent ou lecteur de flux.

Statistique des dernière mise a jours

Sur la page STATS se trouve, l'heurs des dernières synchronisations de flux sur le serveur. De temps a autre cloudflare challenge avec un captcha, il sera donc impossible de faire la synchronisation, c'est en générale temporaire. Nous n'avons pour l'instant aucune solution a se problème.
